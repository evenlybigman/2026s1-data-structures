### 1. clock()을 이용한 실행 시간 측정

``` c
#include <stdio.h>
#include <time.h>

int main() {
	clock_t start, stop;
	double duration;
	long long sum = 0;
	start = clock();
	for (long long i = 0; i < 1000000000; i++) {
		sum += i;
	}
	stop = clock();
	duration = (double)(stop - start) / CLOCKS_PER_SEC;
	printf("sum의 값: %lld\n", sum);
	printf("수행시간은 %lf초입니다.\n",duration);
}

```

만약에 for (long long i = 0; i < 1000000000; i++) ; 이런식으로 쓰게 되면 0.00초로 나오게 되는데, 이는 컴파일러가

프로그램 밖에서 관찰 가능하지 않는 부분은 컴파일 과정에서 삭제해버리기 때문이다.

### 2. time()을 이용한 실행 시간 측정

``` c
#include <stdio.h>
#include <time.h>

int main() {
	time_t start, stop;
	double duration;
	long long sum = 0;
	start = time(NULL);
	for (long long i = 0; i < 1000000000; i++) {
		sum += i;
	}
	stop = time(NULL);
	duration = (double)difftime(stop, start);
	printf("sum의 값: %lld\n", sum);
	printf("수행시간은 %lf초입니다.\n", duration);
}

```

time(NULL)은 Unix 타임 스탬프를 반환한다. (Unix 타임 스탬프는 1970년 1월 1일 00:00:00 (UTC) 부터 현재까지의 초를 정수로 나타낸 것)

time_t 타입은 내부적으로 long 또는 longlong이다.

difftime(time1,time2)는 1의 시간에서 2의 시간을 뺀 값을 반환한다.

time()은 초 단위로 세기 때문에 짧은 시간은 무조건 0.00초로 나온다. 이후로 1초 단위로 바뀐다.

때문에 정확한 시간을 측정할 땐 clock()을, 대략적인 시간을 측정할 땐 time()을 사용한다.

# 재귀 함수

### 재귀 함수를 짜는 것의 핵심

1. 시그니쳐 함수를 찾는다.
   - 이것은 무엇을 받아서 무엇을 반환하는가?
2. 기저 조건을 찾는다.
   - 값이 가장 작을 때 무엇을 리턴하는가?
3. 재귀 조건을 짠다.
   - n을 해결하기 위해 n-1에선 어떤 일이 일어나는가? 이 때 **n-1은 이미 해결되었다고 가정하고 짠다.**

### 팩토리얼

```c
int factorial(int n) {
	if (n == 1) return 1;
	else return n * factorial(n - 1);
}
```

return 값에 식이 올 수 있다. 때문에 factorial(5)가 실행되면

5 * factorial(4) 값이 리턴 된다. 이 factorial(4)는 또 4 * factorial(3)이 되고 factorial(1)까지 반복 된다.

factorial(1)일 때 1이 반환되고 factorial(5)의 최종 반환 값은 5 * 4 * 3 * 2 * 1이 반환되며 120이 반환된다.

반복문을 이용한 팩토리얼은 다음과 같다.

``` c
int main() {
	int result = 1;
	int n = 5;
	for (int n = 5; n > 0; n--) {
		result *= n;
	}
}
```

### X의 n제곱 값을 구하는 함수

처음 짠 코드

``` c
double power(x, n) {
	if (n == 0) return 1;
	return x * power(x, n - 1);
}
```

이 함수의 빅오표기식은 O(n)이다.  

하지만 다음과 같이 코드를 짜면 O(log n)로 줄일 수 있다.

``` c
double power(x, n) {      
	if (n == 0) return 1;
	else if (n % 2 == 0) return power(x * x, n / 2);
	else return x * power(x * x ,(n - 1) / 2);
}
```

빅오 표기법에서 밑을 생략하는 이유

저 코드에서 power함수는 log 2 n번 호출된다. 하지만 빅오에서는 밑을 생략하는데 그 이유는 빅오에서는 상수를 생략하기 때문이다. log 2 n은 상용로그로 바꾸면 log 10 n / log 10 2이다. 여기서 log 10 2는 상수이기 때문에 n의 값을 보는 빅오에서 생략하는 것이다.

반복문으로 짠 코드는 이렇다.

``` c
int power(int x, int n) {
    int result = 1;

    if (n == 0) return 1;
    else if (n % 2 == 0) {
        result = x;
        for (int i = 0; i < n/2; i++) {
            result *= x * x;   // x² 를 n/2번 곱함
        }
    }
    else {
        result = x;
        for (int i = 0; i < (n-1)/2; i++) {
            result *= x * x;   // x² 를 (n-1)/2번 곱함
        }
    }
    return result;
}
```

 ### 피보나치 수열

``` c
int fibonacci(int n) {
	if (n <= 1) return n;
	else return fibonacci(n - 2) + fibonacci(n - 1);
}
```

피보나치 수열은 리턴값이 두번의 재귀 호출인데 이는 n값이 커질수록 호출 값이 기하급수적으로 커지게 됨을 의미한다.

예를 들어 fibonacci(16)은 fibonacci(14)와 fibonacci(15)를 호출하는데 이 fibonacci(14)는 fibonacci(12) + fibonacci(13) 이런 식으로 호출이 계속 된다.

반복문으로 작성한 피보나치 수열은 다음과 같다.

``` c
int fibonacci(int n) {
	int first = 0, second = 1, next = 0;
	if (n <= 1) return n;
	for (int i = 2; i <= n; i++) {
		next = first + second;
		first = second;
		second = next;
	}
	return next;
}
```

n = 0 ,1 일 때는 값이 n이기 때문에 n을 리턴 한다. n이 2 이상일 때 f(n)의 합을 실행한다.

 ### 하노이 탑

하노이 탑의 핵심

n개의 탑을 옮기는 하노이 탑의 알고리즘은 다음과 같다.

1. n-1(n보다 한 칸 위에 있는 탑)의 탑을  임시 공간으로 옮긴다.
2. n번째 탑(가장 아래 있는 탑)을 목적지로 옮긴다.
3. 1에서 임시 공간으로 옮겼던 (n-1)을 다시 목적지로 옮긴다.

n이 몇 개든 이 알고리즘을 통해서 진행된다.

``` c
#include <stdio.h>

void hanoi_tower(int n, char start, char tmp, char end) {
	if (n <= 0) return;
	else if (n == 1) {
		printf("원판 %d를 %c에서 %c로 옮긴다.\n",n, start, end);
	}
	else {
		hanoi_tower(n - 1, start, end, tmp);
		printf("원판 %d를 %c에서 %c로 옮긴다.\n",n, start, end);
		hanoi_tower(n - 1, tmp, start, end);
	}
}

int main() {
	hanoi_tower(4, 'a', 'b', 'c');
}
```
